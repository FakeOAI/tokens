# 高阶玩法

## 指定分组调度 Token

1. 程序内置的默认分组：`free`、`plus`、`pro`、`team`、`max`、`business`、`ultra`
2. **添加Token** 的时候也可以自定义分组
3. 调用接口的时候都是从数据库抽取 `随机分组` 的 token 进行请求的，如果你在调用模型的时候需要指定程序选取何种分组的 token，你可以通过调整模型名称进行分组的指定，**模型名称** 的格式为：`模型名称|[分组,...]`，比如：
   - `gpt-4o|[free]`：调用此模型只会从数据库中轮训 `free` 分组的 token
   - `gpt-4o|[plus,pro]`：调用此模型只会从数据库中轮训 `plus` 和 `pro` 分组的 token
   - `gpt-4o|[custom]`：调用此模型只会从数据库中轮训 `custom` 分组的 token

## 模型参数

模型参数是指可以在模型名称后添加平台支持的参数，通过这种方式可以灵活地通过模型名称来控制模型的功能和表现。

- 格式：`模型名称-[参数1]-[参数2]-...`
- 参数无前后顺序的要求，但必须在对应平台支持的合法取值范围内

例如：`sora_video2` 支持 `10s`、`15s`、`25s`、`pro`、`hd`等参数，那么你可以通过模型参数来指定模型的生成能力，例如：

- `sora_video2-10s-15s`：❌ 参数 '10s' 和 '15s' 不能同时使用
- `sora_video2-30s`：❌ 参数 '30s' 不存在
- `sora_video2-15s`：✅
- `sora_video2-25s-pro`：✅
- `sora_video2-pro-25s`：✅
- `sora_video2-hd-25s`：✅
